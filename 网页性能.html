1、网页、 减少 DNS查询次数，使用域名最好不要超过3个。 减少 http 请求数量，尤其是移动端，最好不要超过，浏览器一般只处理6个并行请求。 避免页面重定向跳转，包括 301 永久重定向 ，302 临时重定向， redirect 强制重定向 页面初始化时延迟加载未显示的图片或其他资源，页面初始化完成后提前加载预知要加载的资源 减少或避免使用 iframe 减少 DOM 元素数量 缓存 AJAX get 请求 2、服务器 使用 CDN 托管静态资源，速度更快，解决地域网速差别 使用 HTTP 缓存，启用 Etag 标记，添加 cache-control meta 头 启用 GZIP 压缩 3、cookie 减少 cookie 数量和数据大小 使用转用域名加载脚本、图片、样式 文件，避免请求带上cookie 4、样式 样式文件置顶，避免页面加载过程中遇到样式重新渲染 避免使用 css 表达式、import、 filter 滤镜。 5、脚本 js 脚本文件放在页面底部 避免多次访问查询 dom 元素，使用保存dom 的变量后记得设置未 null 释放内存 避免密集绑定事件，使用事件代理代替，通过 target 识别触发事件的元素 使用本地缓存 localStorage 存储数据 6、图片 压缩大图片 合并小图片 单独小图片转成 base64 url 减少请求 呈现图片时避免缩放图片 避免空的图片资源请求 7、文件 删减无用代码，抽取公用代码 模块化的脚本文件部署时合并压缩 大文件分割加载，单个文件避免超过25K
